The Environment

The Environment is possibly the most essential part of any programming language implementation. OPAL is no different in this respect. The Environment is defined as a class, and serves two main purposes:

  • Variable and function definition
  • Scope management

The Environment is instantiated as a class variable of the Interpreter. Explicit usage involves calling it through evaluate() with set or def. More complex internal control over the Environment is necessary for many other structures (closed functions and let statements); this is handled internally by calling Environment methods directly.

102    def runClosed(self, closure: "Environment", logic: callable, *args, **kwargs) -> any:
103        """Extended version of the `runLocal` method which allows for the closure of a function.
104        
105        :param `Environment` closure: A closure `Environment`, should be from `INT.interpreter.CLOSURES`.
106        :param `callable` logic: The internal logic of the function to be run locally.
107        :param `*` args: Positional arguments to be passed to `logic`. 
108        :param `**` kwargs: Keyword arguments to be passed to `logic`.
109        
110        :return: The output of `logic(*args, **kwargs)` in local enclosed scope."""
111
112        self.extend(closure)
113        self.begin_scope()
114
115        try:
116            value = logic(*args, **kwargs)
117        finally:
118            closure.env = self.env[:len(closure)+1]
119            self.end_scope(len(closure))
120
121        return value

164    def cleanup(self, var: str, scope: int = None) -> None:
165        """Basic garbage collection for the closure environments attached to functions.
166
167        When a function or other `Closable` object is removed from the environment, its closure
168        `Environment` persists independently as an entry in `INT.interpreter.CLOSURES` and must be
169        manually removed. `cleanup()` is therefore run whenever a variable is removed from the `Environment`, 
170        to check if it is a `Closable`; if so, it removes its entry.
171        
172        :param `str` var: The variable name to be checked.
173        :param (optional) `int` scope: If provided, begins search at given scope. Otherwise starts from lowest scope."""
174
175        scope = scope or 0
176
177        # Get the current variable; if it is a closable, remove its closure
178        current = self.env[scope][var].getVal(); isinstance(current, DAT.Closable) and INT.interpreter.CLOSURES.pop(current.id)

181    def set(self, var: list|str, val: any, scope: int = None) -> None: 
182        """Assign a value to a variable.
183
184            (set a 1)
185            (set (float b) 3.14)
186        
187        The core function of the `Environment`, nearly every operation in the language involves this method
188        to some degree. `set()` takes a variable declaration and creates a `Latent` or `Strict` variable object
189        containing the provided value. A few points to note:
190        
191        - In previous implementations, `set()` would evaluate automatically`val` before binding. This has been
192        deprecated in favor of the increased flexibility of allowing caller to choose whether and how to evaluate.
193            - The built-in function `set` (e.g. `(set a 1)`) **does** evaluate `val`.
194        - `Strict` objects are only created if a type is specified in the passed `var`, i.e. `(int a)`. Otherwise
195        `Latent` objects are the default.
196        - In keeping with the first point, **`set()` does not look up the passed variable**. If the caller wants
197        `set()` to update a previously declared variable (if it exists), the output of `Environment.find_scope(var)` 
198        should be explicitly passed to `scope`. This will then be used if the variable exists, else lowest scope.
199        
200        :param `list|str` var: The variable name. If passed a list, the first element must be a type and the 
201        second must be the variable name. The variable object will be `Strict` with the type set to that provided.
202        Otherwise, the variable will be `Latent` and the type determined by the bound value.
203        :param `any` val: A value to be bound to the variable."""
204
205        scope = scope or 0
206        name = var[1] if isinstance(var, list) else var
207
208        # Set currentVar
209        currentVar = self.env[scope].get(name)
210
211        # Variable is either `Strict`, or `Latent`/undeclared
212        if isinstance(currentVar, DAT.Strict): currentVar.setVal(val)
213        else: self.env[scope][name] = DAT.Variable(var, val)

218        
219            (del x)
220
221        :param `str` var: The name of the variable to be deleted.
222

229        self.cleanup(var, scope)

235

244        (no type included, variable name only) or `Strict` (type included in a tuple with the variable name).
245        :param `list` body: The function body to be evaluated on call."""
246
247        self.set(name, DAT.Function(name, parameters, body))
